## ğŸ” 1. SEO Fundamentals: How Search Engines Work

Search engines discover your content by:

1. **Crawling**: Bots follow links to find pages. In Next.js, if nested dynamic routes are missing or not set up correctly, important pages might be skipped.
2. **Indexing**: Bots add pages to the search database. If some areas are protected or robots.txt is not set up right, important content might be missed.
3. **Rendering**: HTML is processed so bots see the same content as users. Relying too much on client-side rendering is a common Next.js issue that can make pages look empty to search engines.
4. **Ranking**: Content is scored for relevance and quality. If your content structure is unclear or hidden in components, even indexed pages may have trouble ranking well in search results.

If any step fails, such as robots blocking a key page or bots not being able to render HTML, your content may never show up in search results.

## ğŸ§  2. Rendering Strategies That Matter in Next.js

### ğŸ”¹ Static Site Generation (SSG)

Best for marketing pages, blogs, and landing pages. Pre-renders HTML at build time, enabling ultra-fast, reliable SEO crawling.

### ğŸ”¹ Server-Side Rendering (SSR)

SSR is good when content changes often or depends on user data.

### ğŸ”¹ Incremental Static Regeneration (ISR)

ISR updates static pages after deployment without full rebuilds. This is great for large or often updated sites.

SEO depends on rendered HTML, not just JavaScript. Avoid using client-only rendering for important content. Here are two code examples to compare:

**Client-only rendering (bots may see empty content):**

```jsx
<div id="root"></div>

<script src="bundle.js"></script>
```

**Static HTML rendering (bots see real content immediately):**

```jsx
<div>
  <h1>Welcome to Next.js SEO Guide</h1>
  <p>Maximise your organic reach with optimised pages.</p>
</div>
```

Search engines can quickly read and index the real static HTML. In contrast, client-only pages may appear empty or incomplete to them.

## ğŸ“„ 3. Metadata & Social Previews (Next.js Metadata API)

Use Next.js's built-in Metadata API to manage your page metadata. This API generates SEO-relevant HTML tags on the server side before delivering pages to users. It replaces the older `<Head>` component, ensuring SEO tags remain consistent and up to date.

### Static Metadata

```tsx
export const metadata = {
  title: "Awesome App â€” Home",
  description: "A startup that transforms how you..."
}
```

### Dynamic Metadata

For dynamic pages such as blogs or product listings, define an async `generateMetadata` function. This function receives the specific page parameters and generates tailored metadata for each page, providing accurate SEO information and improving search appearance.

```tsx
export async function generateMetadata({ params }) {
  const post = await fetchPost(params.slug);
  return {
    title: post.title,
    description: post.summary,
    openGraph: { ... },
  };
}
```

Test live previews with Twitter Card Validator or Facebook Sharing Debugger to check titles, descriptions, and images.

### Best Practices

- Unique titles per page
- Keep titles under 60 characters
- Write descriptions that are about 150 to 160 characters long
- Include your target keywords in a natural way

## ğŸ“‘ 4. Sitemaps & Robots: Crawl Control

### ğŸ“Œ Sitemap

Create a dynamic sitemap in `app/sitemap.ts` to list all public URLs, including dynamic routes like `blog/[slug]`.

```tsx
export default function sitemap() {
  return [
    { url: "https://yourdomain.com/", lastModified: new Date() },
    // more URLs...
  ];
}
```

### ğŸ“Œ Robots.txt

Create `app/robots.ts` to set rules for bot crawling. For example, block private routes, allow public ones, and link to your sitemap.

Search engines will not index pages that robots.txt blocks. Use this file to protect sensitive areas, such as admin panels.

## ğŸ§  5. Structured Data (JSON-LD)

Structured data helps search engines comprehend your content, not just read it. This is how Google creates rich snippets, such as article previews, breadcrumbs, and FAQ blocks.

### Common Schemas

- Website
- Breadcrumb
- Article
- FAQ
- Organization

Add JSON-LD with a library like next-seo or a shared component. Always validate using Rich Results Test.

## âš¡ 6. Performance & Core Web Vitals

Search engines use performance as a ranking factor. Next.js has features to help if you use them intentionally.

### Important Metrics (Core Web Vitals)

- **LCP**: Measures how quickly the main content loads, such as the hero image or main headline.
- **CLS**: Whether the page stays stable during loading or if content shifts around. Clicks or taps get visible feedback.

### Improve Performance

- Using `next/image` for optimised images
- Preloading fonts with `next/font`
- Avoiding large JS bundles
- Avoiding layout shifts

## ğŸ”— 7. Internal Linking & URL Structure

Search engines follow links. So:

- Create clean, descriptive URLs, such as `/blog/nextjs-seo`, rather than `/post?id=123`
- Use internal links to connect related content
- Treat SEO as an ongoing effort after launch
- Always designate your preferred page versions
